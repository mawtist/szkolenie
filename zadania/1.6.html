Moduł 1, Temat 6: Moduły w JavaScript (Import / Export)

Po co nam moduły? Zamiast pisać 2000 linijek kodu w jednym pliku, możemy stworzyć osobny plik np. matematyka.js (tylko do obliczeń), osobny interfejs.js (do przycisków) i połączyć je ze sobą w głównym pliku app.js.

Wymagany Live Server! Moduły nie zadziałają, jeśli po prostu klikniesz plik HTML na dysku dwa razy. Musisz otworzyć projekt w edytorze (np. VS Code) i użyć rozszerzenia takiego jak "Live Server", aby oszukać przeglądarkę, że strona działa w prawdziwym internecie.

Schemat: Jak pliki komunikują się ze sobą za pomocą Export i Import

1. EXPORT (Wysyłanie danych z pliku)

Wyobraźmy sobie, że stworzyliśmy plik o nazwie matematyka.js. Chcemy, aby inne pliki mogły z niego korzystać. Musimy użyć słowa export.

// --- Zawartość pliku matematyka.js --- // Sposób 1: Export nazwany (Named export) - stawiamy słowo 'export' przed zmienną lub funkcją. export const PI = 3.14159; export function dodaj(a, b) { return a + b; } // Sposób 2: Export domyślny (Default export) - może być TYLKO JEDEN w całym pliku! // Zazwyczaj eksportujemy w ten sposób najważniejszą rzecz w pliku (np. główną Klasę). class Kalkulator { /* kod klasy... */ } export default Kalkulator;

2. IMPORT (Odbieranie danych w innym pliku)

Teraz tworzymy nasz główny plik aplikacji, np. app.js. Chcemy w nim użyć rzeczy z pliku matematycznego.

// --- Zawartość pliku app.js --- // Importowanie rzeczy nazwanych odbywa się w klamrach { }. Nazwy muszą się zgadzać! import { PI, dodaj } from './matematyka.js'; // Importowanie rzeczy domyślnych (Default) odbywa się BEZ klamer. // Możemy ją tu nazwać jak chcemy, ale dobrym zwyczajem jest trzymać się oryginału. import Kalkulator from './matematyka.js'; console.log( "Liczba PI to:", PI ); console.log( "Wynik dodawania:", dodaj(5, 10) ); let mojKalkulator = new Kalkulator();

3. PODPIĘCIE DO HTML

Aby to wszystko zadziałało na naszej stronie internetowej, w pliku HTML musimy podpiąć tylko nasz GŁÓWNY plik (np. app.js) i koniecznie dodać atrybut type="module". Przeglądarka sama "dociągnie" resztę plików dzięki importom.

<!-- Zawartość pliku index.html --> <body> <h1>Moja Aplikacja</h1> <!-- Atrybut type="module" mówi przeglądarce: "Hej, w tym skrypcie będą importy i eksporty!" --> <script type="module" src="app.js"></script> </body>
